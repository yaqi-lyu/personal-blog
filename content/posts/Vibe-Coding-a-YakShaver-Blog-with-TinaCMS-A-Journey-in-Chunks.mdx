Starting with the Foundation: TinaCloud + Vercel
------------------------------------------------

First things first: I needed to get something running. I headed over to TinaCMS and set up a TinaCloud-hosted project, connecting it with Vercel for deployment. The beauty of this combo? TinaCMS handles the content management backend in the cloud, while Vercel takes care of the hosting and deployment. No server management, no backend complexityâ€”just pure focus on building.

The setup was surprisingly smooth. A few clicks, some OAuth connections, and boomâ€”I had a starter template deployed and a CMS dashboard ready to go. That instant gratification hit different. 




The AI Code Archaeology Phase
-----------------------------


Here's where the vibe coding really started. Instead of diving into the starter template blindly, I did something that felt almost like cheating (but isn't): I fed the entire codebase to AI and asked it to map out the structure and architecture for me.



The AI came back with a beautiful breakdown: where the TinaCMS schema lived, how the page components were structured, where the content queries happened, how the build process worked. I took all of this and turned it into a context documentâ€”basically a living map of the codebase that I could reference (and have AI reference) for every coding task moving forward.

This became my north star. Instead of constantly re-explaining the project structure to AI or forgetting how things connected, I had this context ready to paste in. "Here's the architecture, now help me add this feature" became my standard opening move.



## Dreaming Up the Sections



With my foundation solid and my AI context ready, I started vibing on what the YakShaver blog should actually contain. I sketched out the sections I wanted:

*
  A hero section that immediately tells visitors what YakShaver is about
* A featured projects showcase
* A blog post listing
* Maybe a newsletter signup



Nothing revolutionary, but each section needed its own personality, its own vibe.



## Building in Chunks: The Homepage Blocks



This is where TinaCMS really shines, and where my chunk-by-chunk approach felt \*right\*.

TinaCMS uses a blocks-based systemâ€”each section of your page is a reusable, editable block. So instead of trying to build the entire homepage in one marathon session, I built one block at a time:



**Chunk 1: The Hero Block**


I started with the hero. It generated the schema definition, the component, and showed me how to register it. I tweaked the styling until it felt right, deployed, and checked it in the CMS. Satisfying.



**Chunk 2: The Featured Projects Block**


Next up: showcasing projects. Same processâ€”feed the context to AI, describe what I wanted (a grid of project cards with images, titles, and descriptions), and let it scaffold the block structure. I iterated on the layout, made the cards responsive, added some hover effects. Another chunk done.



**Chunk 3: Blog Listing Block**


For the blog posts, I needed to query the content collection. AI helped me set up the GraphQL query to fetch recent posts, create the listing component, and make each post card look clean. The beautiful thing? TinaCMS auto-generates the query system, so I was just connecting dots.

Each chunk took maybe 30-60 minutes. Build, test, deploy, admire. Then take a break. Come back refreshed for the next chunk.









## What I Learned



Building this way taught me that modern web development doesn't have to be overwhelming. You don't need to hold the entire system in your head. You don't need to build everything at once.

TinaCMS gave me the structure. Vercel gave me the instant deploys. AI gave me the velocity. But the vibe? That was all about trusting the process, building in chunks, and letting each piece inform the next.

The YakShaver blog came together not because I had a perfect plan, but because I vibed my way through it, one satisfying chunk at a time.
